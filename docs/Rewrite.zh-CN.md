# 转写

本文档介绍 Voxt 的转写功能：它做什么、和普通转录 / 翻译有什么区别，以及如何配置。

## 功能概览

转写是 Voxt 里的“语音驱动 Prompt / 改写”能力。

默认快捷键：`fn+control`

它会先把你的语音识别成“指令”，再调用当前配置的转写模型来：

- 改写当前选中文本
- 根据口述要求直接生成新文本
- 在不适合直接写回输入框时，用答案卡片稳定展示结果

## 两种主要使用方式

### 改写选中文本

如果当前有选中文本，你的口述会作为“改写要求”应用到这段原文上。

例如：

- “帮我精简一下，语气自然一点”
- “改成更正式的语气”
- “整理成要点列表”

### 根据语音直接生成

如果当前没有选中文本，你的口述本身就会作为 Prompt。

例如：

- “帮我写一段 200 字的自我介绍”
- “写一段发给老板的项目进展”

## 模型与 Provider

转写跟随主窗口 `模型` 页中的转写 provider / model 配置。

当前支持：

- 本地 Custom LLM
- Remote LLM

ASR 模型只负责把你的语音转成指令文本；真正生成 / 改写结果时，走的是转写模型链路。

## 提示词

转写使用独立的“转写提示词”。

- 可以在主窗口里自定义
- 你的口述指令会作为转写请求的一部分
- 如果当前有选中文本，原文会作为 source 一起传入

在合适的场景下，Voxt 也会先对口述指令做一次常规增强，再进入转写阶段。

## 应用增强与转写

开启 `应用增强（App Enhancement）` 后，转写也可以变成“按上下文切换”的能力。

<video src="https://storage.actnow.dev/common/voxt/voxt-app-branch-demo.mp4" controls preload="none" width="100%"></video>

- Voxt 可以根据当前 App 或浏览器 URL 自动切换不同的 Prompt 和规则
- 同一个转写快捷键，在 IDE、聊天工具、邮件、文档场景下可以表现不同
- 这很适合做不同上下文下的语气、格式、术语差异化

`应用增强` 不是替代转写本身，而是改变转写实际使用的提示词路由和上下文。

## 转写答案卡片

当不适合直接写回输入框时，转写可以把结果稳定展示在答案卡片里。

这在下面这些场景里很有用：

- 当前目标不可写
- 生成内容较长
- 你想先看结果再决定怎么用

## 使用建议

- 明确说明风格、长度、格式
- 如果是改写选中文本，重点说“要怎么改”
- 如果是直接生成，重点说“要写什么”和“有哪些限制”

## 注意事项

- 转写和普通转录、翻译是独立流程
- 底层可以复用相同的本地 / 远端 LLM，但提示词和输出行为不同
- 结果质量会明显受到模型选择和口述指令清晰度影响
